# РАДАР — Система предиктивного мониторинга технического состояния насосного оборудования на основе алгоритмов машинного обучения


[![Python](https://img.shields.io/badge/python-3.11-blue.svg)](https://www.python.org/downloads/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

**РАДАР** (РАДАР — Раннее Автоматическое Детектирование Аномалий и Рисков) — система предиктивного мониторинга промышленного оборудования с веб-интерфейсом, Telegram-уведомлениями и Push-оповещениями.
**Демо репозиторий**
## 📋 Возможности

- **Мониторинг в реальном времени** — отслеживание вибрации, температуры, давления, тока двигателя
- **Предиктивная аналитика** — расчёт индекса здоровья оборудования и прогноз остаточного ресурса
- **Обнаружение аномалий** — ИИ-детектор на базе Isolation Forest (scikit-learn)
- **Веб-дашборд** — графики, история событий, экспорт отчётов (PDF)
- **Уведомления**:
  - Telegram-бот с командами `/status`, `/health`, `/events`, `/reset`
  - Web Push notifications для браузера
- **Безопасность**:
  - HTTPS по умолчанию (автогенерация сертификатов)
  - Блокировка при множественных неудачных входах
  - Хеширование паролей (bcrypt)
- **Контейнеризация** — готовая сборка через Docker и docker-compose
- **База данных** — PostgreSQL (продакшен) или SQLite (разработка)

## 🚀 Быстрый старт

### Вариант 1: Docker Compose (рекомендуется)

```bash
# Клонировать репозиторий
git clone <repository-url> && cd radar

# Запустить сервисы
docker-compose up -d

# Открыть дашборд
https://localhost:5000
```

**Учётные данные по умолчанию:**
- Логин: `admin`
- Пароль: задайте через ENV

### Вариант 2: Локальный запуск

```bash
# Создание виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# или .venv\Scripts\activate  # Windows

# Установка зависимостей
pip install -r requirements.txt

# Запуск веб-интерфейса
python main.py --web

# Или консольный режим
python main.py --mode bearing --rate 0.0004
```

## ⚙️ Конфигурация

### Переменные окружения

| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| `DATABASE_URL` | PostgreSQL connection string | (SQLite) |
| `SECRET_KEY` | Ключ сессий Flask | (автогенерация) |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота | — |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений | — |
| `DB_RETENTION_HOURS` | Срок хранения записей в БД | `168` (7 дней) |
| `MAX_LOGIN_ATTEMPTS` | Максимум попыток входа | `5` |
| `LOCKOUT_SECONDS` | Время блокировки после неудачного входа | `300` |
| `SESSION_TIMEOUT` | Таймаут сессии | `1800` (30 мин) |

### Сценарии отказа

Система поддерживает симуляцию различных типов неисправностей:

```bash
# Отказ подшипника
python main.py --web --mode bearing

# Отказ уплотнения
python main.py --web --mode seal

# Перегрузка двигателя
python main.py --web --mode overload
```

## 🔧 API и интеграция

### Telegram-бот команды

| Команда | Описание |
|---------|----------|
| `/status` | Текущие показания датчиков |
| `/health` | Индекс здоровья и прогноз отказа |
| `/events` | Последние события |
| `/reset` | Сброс детектора аномалий (только авторизованный chat) |
| `/help` | Справка по командам |

### Web Push

Для включения Push-уведомлений:
1. Откройте дашборд в браузере
2. Разрешите уведомления при запросе
3. Настройте VAPID-ключи в `.vapid_keys.json`

## 🔒 Безопасность

- **HTTPS** обязателен в продакшене (сертификаты генерируются автоматически)
- **Пароли** хранятся в хешированном виде (bcrypt, 12 раундов)
- **Защита от брутфорса** — блокировка после N неудачных попыток
- **Сессионные токены** — ограниченный срок жизни
- **Валидация входных данных** — защита от XSS и инъекций
